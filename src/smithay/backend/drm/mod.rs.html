1//! This module represents abstraction on top the linux direct rendering manager api (drm).
2//!
3//! ## [`DrmDevice`]
4//!
5//! A device exposes certain properties, which are directly derived
6//! from the *device* as perceived by the direct rendering manager api (drm). These resources consists
7//! out of:
8//! - [`connectors`](drm::control::connector) represents a port on your computer, possibly with a connected monitor, TV, capture card, etc.
9//! - [`encoder`](drm::control::encoder) encodes the data of connected crtcs into a video signal for a fixed set of connectors.
10//!   E.g. you might have an analog encoder based on a DAG for VGA ports, but another one for digital ones.
11//!   Also not every encoder might be connected to every crtc.
12//! - [`framebuffer`] represents a buffer you may display, see `DrmSurface` below.
13//! - [`plane`] adds another layer on top of the crtcs, which allow us to layer multiple images on top of each other more efficiently
14//!   then by combining the rendered images in the rendering phase, e.g. via OpenGL. Planes have to be explicitly used by the user to be useful.
15//!   Every device has at least one primary plane used to display an image to the whole crtc. Additionally cursor and overlay planes may be present.
16//!   Cursor planes are usually very restricted in size and meant to be used for hardware cursors, while overlay planes may
17//!   be used for performance reasons to display any overlay on top of the image, e.g. the top-most windows.
18//!   Overlay planes may have a bunch of weird limitation, that you cannot query, e.g. only working on round pixel coordinates.
19//!   You code should never rely on a fixed set of overlay planes, but always have a fallback solution in place.
20//! - [`crtc`]s represent scanout engines of the device pointer to one framebuffer.
21//!   Their responsibility is to read the data of the framebuffer and export it into an "Encoder".
22//!   The number of crtc's represent the number of independent output devices the hardware may handle.
23//!
24//!  On modern graphic cards it is better to think about the `crtc` as some sort of rendering engine.
25//!  You can only have so many different pictures, you may display, as you have `crtc`s, but a single image
26//!  may be put onto multiple displays.
27//!
28//! The main functionality of a `Device` in smithay is to give access to all these properties for the user to
29//! choose an appropriate rendering configuration. What that means is defined by the requirements and constraints documented
30//! in the specific device implementations. The second functionality is the creation of a `Surface`.
31//! Surface creation requires a `crtc` (which cannot be the same as another existing `Surface`'s crtc), a plane,
32//! as well as a `Mode` and a set of `connectors`.
33//!
34//! smithay does not make sure that `connectors` are not already in use by another `Surface`. Overlapping `connector`-Sets may
35//! be an error or result in undefined rendering behavior depending on the `Surface` implementation.
36//!
37//! ## [`DrmSurface`]
38//!
39//! A surface is a part of a `Device` that may output a picture to a number of connectors. It pumps pictures of buffers to outputs.
40//!
41//! On surface creation a matching encoder for your `connector` is automatically selected, if one exists(!),
42//! which means you still need to check your configuration beforehand, even if you do not need to provide an encoder.
43//!
44//! A surface consists of one `crtc` that is rendered to by the user. This is fixed for the `Surface`s lifetime and cannot be changed.
45//! A surface also always needs at least one connector to output the resulting image to as well as a `Mode` that is valid for the given connectors.
46//!
47//! The state of a `Surface` is double-buffered, meaning all operations that chance the set of `connector`s or their `Mode` are stored and
48//! only applied on the next commit. `Surface`s do their best to validate these changes, if possible.
49//!
50//! A commit/page_flip may be triggered to apply the pending state.
51//!
52//! ## Rendering
53//!
54//! The drm infrastructure makes no assumptions about the used renderer and does not interface with them directly.
55//! It just provides a way to create framebuffers from various buffer types (mainly `DumbBuffer`s and hardware-backed gbm `BufferObject`s).
56//!
57//! Buffer management and details about the various types can be found in the [`allocator`-Module](crate::backend::allocator) and
58//! rendering abstractions, which can target these buffers can be found in the [`renderer`-Module](crate::backend::renderer).
59//!
60//! ### Hardware composition
61//!
62//! The [`DrmCompositor`](crate::backend::drm::compositor::DrmCompositor) provides a simplified way to utilize drm planes for
63//! using hardware composition.
64//! See the [`compositor`] module docs for more information on that topic.
65//!
66//! ## [`DrmNode`]
67//!
68//! A drm node refers to a drm device and the capabilities that may be performed using the node.
69//! Generally [`DrmNode`] is primarily used by clients (such as the output backends) which need
70//! to allocate buffers for use in X11 or Wayland. If you need to do mode setting, you should use
71//! [`DrmDevice`] instead.

126/// A set of planes as supported by a crtc
127#[derive(Debug, Clone)]
128pub struct Planes {
129    /// The primary plane(s) of the crtc
130    pub primary: Vec<PlaneInfo>,
131    /// The cursor plane(s) of the crtc, if available
132    pub cursor: Vec<PlaneInfo>,
133    /// Overlay planes supported by the crtc, if available
134    pub overlay: Vec<PlaneInfo>,
135}
136
137/// Info about a single plane
138#[derive(Debug, Clone)]
139pub struct PlaneInfo {
140    /// Handle of the plane
141    pub handle: plane::Handle,
142    /// Type of the plane
143    pub type_: PlaneType,
144    /// z-position of the plane if available
145    pub zpos: Option<i32>,
146    /// Formats supported by this plane
147    pub formats: FormatSet,
148    /// Recommended plane size in order of preference
149    pub size_hints: Option<Vec<Size<u16, Physical>>>,
150}
151
152fn planes(
153    dev: &(impl DevPath + ControlDevice),
154    crtc: &crtc::Handle,
155    has_universal_planes: bool,
156) -> Result<Planes, DrmError> {
157    let mut primary = Vec::with_capacity(1);
158    let mut cursor = Vec::new();
159    let mut overlay = Vec::new();
160
161    let planes = dev.plane_handles().map_err(|source| {
162        DrmError::Access(AccessError {
163            errmsg: "Error loading plane handles",
164            dev: dev.dev_path(),
165            source,
166        })
167    })?;
168
169    let resources = dev.resource_handles().map_err(|source| {
170        DrmError::Access(AccessError {
171            errmsg: "Error loading resource handles",
172            dev: dev.dev_path(),
173            source,
174        })
175    })?;
176
177    for plane in planes {
178        let info = dev.get_plane(plane).map_err(|source| {
179            DrmError::Access(AccessError {
180                errmsg: "Failed to get plane information",
181                dev: dev.dev_path(),
182                source,
183            })
184        })?;
185        let filter = info.possible_crtcs();
186        if resources.filter_crtcs(filter).contains(crtc) {
187            let zpos = plane_zpos(dev, plane).ok().flatten();
188            let type_ = plane_type(dev, plane)?;
189            let formats = plane_formats(dev, plane)?;
190            let size_hints = plane_size_hints(dev, plane)?;
191            let plane_info = PlaneInfo {
192                handle: plane,
193                type_,
194                zpos,
195                formats,
196                size_hints,
197            };
198            match type_ {
199                PlaneType::Primary => {
200                    primary.push(plane_info);
201                }
202                PlaneType::Cursor => {
203                    cursor.push(plane_info);
204                }
205                PlaneType::Overlay => {
206                    overlay.push(plane_info);
207                }
208            };
209        }
210    }
211
212    Ok(Planes {
213        primary,
214        cursor: if has_universal_planes { cursor } else { Vec::new() },
215        overlay: if has_universal_planes { overlay } else { Vec::new() },
216    })
217}

219fn plane_type(dev: &(impl ControlDevice + DevPath), plane: plane::Handle) -> Result<PlaneType, DrmError> {
220    let props = dev.get_properties(plane).map_err(|source| {
221        DrmError::Access(AccessError {
222            errmsg: "Failed to get properties of plane",
223            dev: dev.dev_path(),
224            source,
225        })
226    })?;
227    let (ids, vals) = props.as_props_and_values();
228    for (&id, &val) in ids.iter().zip(vals.iter()) {
229        let info = dev.get_property(id).map_err(|source| {
230            DrmError::Access(AccessError {
231                errmsg: "Failed to get property info",
232                dev: dev.dev_path(),
233                source,
234            })
235        })?;
236        if info.name().to_str().map(|x| x == "type").unwrap_or(false) {
237            return Ok(match val {
238                x if x == (PlaneType::Primary as u64) => PlaneType::Primary,
239                x if x == (PlaneType::Cursor as u64) => PlaneType::Cursor,
240                _ => PlaneType::Overlay,
241            });
242        }
243    }
244    unreachable!()
245}

247fn plane_zpos(dev: &(impl ControlDevice + DevPath), plane: plane::Handle) -> Result<Option<i32>, DrmError> {
248    let props = dev.get_properties(plane).map_err(|source| {
249        DrmError::Access(AccessError {
250            errmsg: "Failed to get properties of plane",
251            dev: dev.dev_path(),
252            source,
253        })
254    })?;
255    let (ids, vals) = props.as_props_and_values();
256    for (&id, &val) in ids.iter().zip(vals.iter()) {
257        let info = dev.get_property(id).map_err(|source| {
258            DrmError::Access(AccessError {
259                errmsg: "Failed to get property info",
260                dev: dev.dev_path(),
261                source,
262            })
263        })?;
264        if info.name().to_str().map(|x| x == "zpos").unwrap_or(false) {
265            let plane_zpos = match info.value_type().convert_value(val) {
266                drm::control::property::Value::UnsignedRange(u) => Some(u as i32),
267                drm::control::property::Value::SignedRange(i) => Some(i as i32),
268                // A range from [0,1] will be interpreted as Boolean in drm-rs
269                // TODO: Once that has been changed we can remove this special handling here
270                drm::control::property::Value::Boolean(b) => Some(b.into()),
271                _ => None,
272            };
273            return Ok(plane_zpos);
274        }
275    }
276    Ok(None)
277}

279fn plane_formats(dev: &(impl ControlDevice + DevPath), plane: plane::Handle) -> Result<FormatSet, DrmError> {
280    // get plane formats
281    let plane_info = dev.get_plane(plane).map_err(|source| {
282        DrmError::Access(AccessError {
283            errmsg: "Error loading plane info",
284            dev: dev.dev_path(),
285            source,
286        })
287    })?;
288    let mut formats = IndexSet::new();
289    for code in plane_info
290        .formats()
291        .iter()
292        .flat_map(|x| DrmFourcc::try_from(*x).ok())
293    {
294        formats.insert(DrmFormat {
295            code,
296            modifier: DrmModifier::Invalid,
297        });
298    }
299
300    if let Ok(1) = dev.get_driver_capability(DriverCapability::AddFB2Modifiers) {
301        let set = dev.get_properties(plane).map_err(|source| {
302            DrmError::Access(AccessError {
303                errmsg: "Failed to query properties",
304                dev: dev.dev_path(),
305                source,
306            })
307        })?;
308        let (handles, _) = set.as_props_and_values();
309        // for every handle ...
310        let prop = handles
311            .iter()
312            .find(|handle| {
313                // get information of that property
314                if let Ok(info) = dev.get_property(**handle) {
315                    // to find out, if we got the handle of the "IN_FORMATS" property ...
316                    if info.name().to_str().map(|x| x == "IN_FORMATS").unwrap_or(false) {
317                        // so we can use that to get formats
318                        return true;
319                    }
320                }
321                false
322            })
323            .copied();
324        if let Some(prop) = prop {
325            let prop_info = dev.get_property(prop).map_err(|source| {
326                DrmError::Access(AccessError {
327                    errmsg: "Failed to query property",
328                    dev: dev.dev_path(),
329                    source,
330                })
331            })?;
332            let (handles, raw_values) = set.as_props_and_values();
333            let raw_value = raw_values[handles
334                .iter()
335                .enumerate()
336                .find_map(|(i, handle)| if *handle == prop { Some(i) } else { None })
337                .unwrap()];
338            if let drm::control::property::Value::Blob(blob) = prop_info.value_type().convert_value(raw_value)
339            {
340                let data = dev.get_property_blob(blob).map_err(|source| {
341                    DrmError::Access(AccessError {
342                        errmsg: "Failed to query property blob data",
343                        dev: dev.dev_path(),
344                        source,
345                    })
346                })?;
347                // be careful here, we have no idea about the alignment inside the blob, so always copy using `read_unaligned`,
348                // although slice::from_raw_parts would be so much nicer to iterate and to read.
349                unsafe {
350                    let fmt_mod_blob_ptr = data.as_ptr() as *const drm_ffi::drm_format_modifier_blob;
351                    let fmt_mod_blob = &*fmt_mod_blob_ptr;
352
353                    let formats_ptr: *const u32 = fmt_mod_blob_ptr
354                        .cast::<u8>()
355                        .offset(fmt_mod_blob.formats_offset as isize)
356                        as *const _;
357                    let modifiers_ptr: *const drm_ffi::drm_format_modifier = fmt_mod_blob_ptr
358                        .cast::<u8>()
359                        .offset(fmt_mod_blob.modifiers_offset as isize)
360                        as *const _;
361                    #[allow(clippy::unnecessary_cast)]
362                    let formats_ptr = formats_ptr as *const u32;
363                    #[allow(clippy::unnecessary_cast)]
364                    let modifiers_ptr = modifiers_ptr as *const drm_ffi::drm_format_modifier;
365
366                    for i in 0..fmt_mod_blob.count_modifiers {
367                        let mod_info = modifiers_ptr.offset(i as isize).read_unaligned();
368                        for j in 0..64 {
369                            if mod_info.formats & (1u64 << j) != 0 {
370                                let code = DrmFourcc::try_from(
371                                    formats_ptr
372                                        .offset((j + mod_info.offset) as isize)
373                                        .read_unaligned(),
374                                )
375                                .ok();
376                                let modifier = DrmModifier::from(mod_info.modifier);
377                                if let Some(code) = code {
378                                    formats.insert(DrmFormat { code, modifier });
379                                }
380                            }
381                        }
382                    }
383                }
384            }
385        }
386    } else if plane_type(dev, plane)? == PlaneType::Cursor {
387        // Force a LINEAR layout for the cursor if the driver doesn't support modifiers
388        for format in formats.clone() {
389            formats.insert(DrmFormat {
390                code: format.code,
391                modifier: DrmModifier::Linear,
392            });
393        }
394    }
395
396    if formats.is_empty() {
397        formats.insert(DrmFormat {
398            code: DrmFourcc::Argb8888,
399            modifier: DrmModifier::Invalid,
400        });
401    }
402
403    trace!(
404        "Supported scan-out formats for plane ({:?}): {:?}",
405        plane,
406        formats
407    );
408
409    Ok(FormatSet::from_formats(formats))
410}

412#[cfg(feature = "backend_gbm")]
413fn plane_has_property(
414    dev: &(impl drm::control::Device + DevPath),
415    plane: plane::Handle,
416    name: &str,
417) -> Result<bool, DrmError> {
418    let props = dev.get_properties(plane).map_err(|source| {
419        DrmError::Access(AccessError {
420            errmsg: "Failed to get properties of plane",
421            dev: dev.dev_path(),
422            source,
423        })
424    })?;
425    let (ids, _) = props.as_props_and_values();
426    for &id in ids {
427        let info = dev.get_property(id).map_err(|source| {
428            DrmError::Access(AccessError {
429                errmsg: "Failed to get property info",
430                dev: dev.dev_path(),
431                source,
432            })
433        })?;
434        if info.name().to_str().map(|x| x == name).unwrap_or(false) {
435            return Ok(true);
436        }
437    }
438    Ok(false)
439}

448fn plane_size_hints(
449    dev: &(impl ControlDevice + DevPath),
450    plane: plane::Handle,
451) -> Result<Option<Vec<Size<u16, Physical>>>, DrmError> {
452    let props = dev.get_properties(plane).map_err(|source| {
453        DrmError::Access(AccessError {
454            errmsg: "Failed to get properties of plane",
455            dev: dev.dev_path(),
456            source,
457        })
458    })?;
459    let (ids, vals) = props.as_props_and_values();
460    for (&id, &val) in ids.iter().zip(vals.iter()) {
461        let info = dev.get_property(id).map_err(|source| {
462            DrmError::Access(AccessError {
463                errmsg: "Failed to get property info",
464                dev: dev.dev_path(),
465                source,
466            })
467        })?;
468        if info.name().to_str().map(|x| x == "SIZE_HINTS").unwrap_or(false) {
469            let size_hints = if let drm::control::property::Value::Blob(blob_id) =
470                info.value_type().convert_value(val)
471            {
472                // Note that property value 0 (ie. no blob) is reserved for potential
473                // future use. Current userspace is expected to ignore the property
474                // if the value is 0
475                if blob_id == 0 {
476                    return Ok(None);
477                }
478
479                let blob_info = drm_ffi::mode::get_property_blob(dev.as_fd(), blob_id as u32, None).map_err(
480                    |source| {
481                        DrmError::Access(AccessError {
482                            errmsg: "Failed to get SIZE_HINTS blob info",
483                            dev: dev.dev_path(),
484                            source,
485                        })
486                    },
487                )?;
488
489                let mut data = Vec::with_capacity(blob_info.length as usize);
490                drm_ffi::mode::get_property_blob(dev.as_fd(), blob_id as u32, Some(&mut data)).map_err(
491                    |source| {
492                        DrmError::Access(AccessError {
493                            errmsg: "Failed to get SIZE_HINTS blob data",
494                            dev: dev.dev_path(),
495                            source,
496                        })
497                    },
498                )?;
499
500                let num_size_hints = data.len() / std::mem::size_of::<drm_plane_size_hint>();
501                let size_hints = unsafe {
502                    std::slice::from_raw_parts(data.as_ptr() as *const drm_plane_size_hint, num_size_hints)
503                };
504                Some(
505                    size_hints
506                        .iter()
507                        .map(|size_hint| Size::from((size_hint.width, size_hint.height)))
508                        .collect(),
509                )
510            } else {
511                tracing::debug!(?plane, "SIZE_HINTS property has wrong value type");
512                None
513            };
514            return Ok(size_hints);
515        }
516    }
517    Ok(None)
